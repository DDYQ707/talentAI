 **1、确定4 个核心角色行为动线：**

**候选人 (Candidate)**

- **旅程**：打开 H5 首页 -> 浏览/搜索/筛选岗位 -> 查看岗位详情 -> 填写基础信息并上传简历（触发 AI 异步解析） -> 提交投递 -> 通过手机号或登录状态查询投递进度（如：HR筛选中、面试安排）。

**HR (招聘专员)**

- **旅程**：登录工作台 -> 手动发布/管理岗位 -> 在简历管理页查看投递 -> **查看 AI 解析结果与人岗匹配评分** -> 筛选合格者 -> 发起面试安排（分配时间与面试官） -> 面试后查看评价 -> 将淘汰人员归档至人才库。

**面试官 (Interviewer)**

- **旅程**：登录工作台 -> 查看今日待面试列表 -> 查看脱敏简历与 **AI 匹配优势/不足** -> **一键生成专属 AI 面试问题** -> 进行线下/视频外置面试 -> 录入多维评分与综合评价 -> 系统自动更新 **AI 人才画像**。

**系统管理员 (Admin)**

- **旅程**：配置企业部门 -> 分配 HR 与面试官的账号权限 -> 配置底层运行参数（如 OSS 密钥、系统公告等）。

**2、明确一期（MVP）的绝对边界**

**简历解析 Agent**：从 PDF/Word/图片中抽取【基本信息、教育经历、工作经历、核心技能】，转化为标准的 JSON 格式入库。

**人岗匹配 Agent**：读取岗位 JD 文本与解析后的简历结构化数据，输出【匹配度 0-100 分】及【优势 3 条、不足 2 条】。

**面试问题生成 Agent**：读取简历与岗位差异，生成【3-5个针对性面试追问】（如：该候选人缺乏高并发经验，生成一道考察其应对流量峰值思路的问题）。

**人才画像 Agent**：综合候选人简历亮点、劣势及面试官的文字评价，生成【一段 200 字左右的综合人才画像总结】。

后续迭代储备：AI 生成/优化岗位 JD、AI 对话式招聘助手、AI 面试官（音视频作弊检测/情绪识别）、AI 薪资建议、AI 决策预测。

**4、微服务职责划分明细表**

 9 个核心模块

1. **`talent-gateway` (API 网关)**：负责所有端点路由转发，集成全局 JWT Token 校验拦截器。
2. **`talent-auth` (认证中心)**：负责 4 类角色（候选人、HR、面试官、管理员）的登录鉴权（基于 BCrypt 加密）及基础 RBAC 权限下发。
3. **`talent-common` (公共组件)**：提供给其他服务引用的基础 Jar 包，包含统一响应体 (`R.java`)、全局异常拦截、分页参数定义及公共工具类。
4. **`talent-job` (岗位与投递)**：负责 HR 发布岗位、H5 岗位浏览搜索、以及投递记录状态机的流转。
5. **`talent-resume` (简历服务)**：负责对接 MinIO 处理文件上传，管理候选人投递时的原始简历信息。
6. **`talent-ai-agent` (AI 核心枢纽)**：集成 LangChain4j 与通义千问 API。提供内部 RPC 接口供其他服务调用（执行简历解析、人岗匹配打分、画像生成、面试题生成）。
7. **`talent-interview` (面试管理)**：管理面试日程安排、面试官分配及多维评分录入。
8. **`talent-talent-pool` (人才储备库)**：管理淘汰流转节点，提供多维度条件搜索。
9. **`talent-analytics` (数据聚合)**：纯查询服务，跨模块聚合首页待办、投递总量及招聘漏斗图。

2.  核心 AI Agent 的触发闭环设计

AI 能力以**异步事件或显式调用**

**触发解析 Agent**：候选人点击“提交投递” -> 文件上传 MinIO 成功 -> 异步调用 `talent-ai-agent` 提取信息 -> 写入 `resume_*` 相关子表。

**触发匹配 Agent**：解析入库完成后 -> 异步读取 `job_info` 与提取出的经历 -> 调用通义千问打分 -> 写入 `ai_match_record`。

**触发面试题 Agent**：面试官在详情页点击【AI 生成面试问题】按钮 -> **同步调用**通义千问 -> 页面展示生成的题目。

**触发画像 Agent**：HR 点击淘汰或录用时 -> 异步整合所有的 `resume_info` 和 `interview_feedback` -> 刷新并保存至 `ai_talent_profile`。
