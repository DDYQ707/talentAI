# 智能招聘与人才画像分析系统 — 功能清单

> **文档说明**：基于仓库代码静态分析（前端 `talent-ai-front` + 后端 `talent-ai-backend` + SQL 快照 `docs/sql/`）。  
> **最后更新**：2026-07-01  
> **范围**：仅列出当前已实现并可演示的功能。

---

## 一、系统概览

| 维度 | 说明 |
|------|------|
| 架构 | 微服务 + API 网关（8080）+ Vue 3 单页前端 |
| 后端 | 9 个可运行服务 + 公共库 `talent-common` |
| 前端 | 单一 SPA，按 `user_type` 路由至候选人 / HR / 面试官 / 管理员四端 |
| 核心业务链 | 登录 → 投递 → AI 解析/匹配 → HR 筛选 → 面试 → Offer → 人才库归档 |
| 数据库 | 7 个业务库，一键初始化 `docs/sql/init_all.sql` |
| 基础设施 | Docker 提供 Redis / Nacos / MinIO；MySQL 使用本机 3306 |

### 微服务一览

| 服务 | 端口 | 说明 |
|------|------|------|
| talent-gateway | 8080 | JWT 鉴权、路由转发、CORS、Sentinel 限流（429） |
| talent-auth | 8081 | 认证、用户、通知、候选人档案 |
| talent-job | 8082 | 岗位、投递、收藏、Offer |
| talent-resume | 8083 | 附件/在线简历、MinIO 存储 |
| talent-ai-agent | 8084 | AI 解析、匹配、画像、助手、知识库、面试题/笔记、简历质量分 |
| talent-interview | 8085 | 面试安排、评价、统计 |
| talent-talent-pool | 8086 | 人才库归档、标签、检索 |
| talent-analytics | 8087 | HR 驾驶舱 Feign 聚合（无独立库） |
| talent-admin | 8088 | 平台管理、大屏、风控、广播、字典、企业审核 |
| talent-common | — | 统一响应 `R<T>`、JWT 工具 |

## 二、基础设施与公共能力

| 功能 | 说明 |
|------|------|
| Docker Compose | `docker/docker-compose.yml`：Redis（6380）、Nacos（8848）、MinIO（9000/9001） |
| 数据库初始化 | `docs/sql/init_all.sql` + 7 个 `talent_*_db.sql` 全量快照（结构 + 演示数据） |
| API 网关鉴权 | `AuthGlobalFilter`：JWT 校验、401 JSON、`X-User-Id` / `X-User-Role` 注入 |
| Sentinel | 网关 429 限流；analytics Feign Fallback |
| Nacos 服务注册 | 各服务 `bootstrap.yml`，命名空间 `talent-ai-dev` |
| Nacos 配置中心 | auth 等模块已接入（如 OTP 配置） |
| 统一响应 `R<T>` | `talent-common` |
| JWT 签发与校验 | 登录签发，网关与各服务校验 |
| Feign 服务间调用 | job / resume / interview / ai / admin / analytics 等模块互联 |
| E2E 冒烟脚本 | `scripts/verify_e2e_chain.ps1` |

## 十二、跨模块业务链路

| 链路 | 说明 |
|------|------|
| 投递 → AI 解析 → 匹配分数回写 | job / resume / ai-agent Feign 联动 |
| 投递 → 系统通知 | auth 通知服务 |
| HR 初筛 → 安排面试 → 评价 → 改状态 | interview ↔ resume |
| 评价完成 → AI 人才画像 | interview → ai-agent |
| 淘汰 / Offer 拒绝 → 人才库归档 | job → talent-pool |
| 面试通过 → Offer 发放 → 候选人确认 | job Offer 全生命周期 |
| 管理员风控下架 → 岗位真实下架 | admin Feign → job internal |
| 公告广播 → 用户通知 | admin Feign → auth broadcast |
